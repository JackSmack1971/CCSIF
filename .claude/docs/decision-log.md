## 2026-07-09 — Architecture audit and remediation pass

Source: `.claude/audits/claude-code-architecture-audit.md` (deterministic, 25 findings) plus manual review of hook scripts and an untracked rule file. Full findings and rationale: `.claude/audits/claude-code-improvement-plan.md`.

- **Removed** the untracked `.claude/rules/System Architecture and Empirically Grounded Self-Improvement in Claude Code v2.1.206.md` (204 lines, no `paths:` scoping, loaded unconditionally every turn). It contained an embedded "migration protocol" instructing autonomous restructuring and git-committing of `.claude/` as self-improvement — operational authority that belongs to the repo's existing approval gates (`control-plane.md`, `operating-constitution.md`), not to an unreviewed document. Relocated the descriptive content to `.claude/docs/claude-code-architecture-reference.md` (not auto-loaded) with the operational sections stripped and a notice that its citations are unverified.
- **Implemented** real logic in the previously-placeholder `pre-tool-use.sh` (now delegates to `.claude/hooks/lib/pre-tool-use-guard.js`, which blocks `Write`/`Edit`/`NotebookEdit`/mutating-`Bash` calls that target a CLAUDE.md Protected Area — secrets, CI/CD workflows, migrations, auth, payments, production config) and `stop.sh` (now runs the deterministic `.claude/scripts/control_plane_check.py` instead of a repo-wide whitespace scan that was noisy under Windows autocrlf). Both previously only echoed a placeholder comment. Note: `stop.sh` cannot run project test/lint/typecheck commands yet — `CLAUDE.md`'s "Source-of-Truth Commands" section is still an unfilled npm-flavored template and this repo has no `package.json`. Fill in real commands there and wire them into `stop.sh` once they're known.
- **Fixed** `architecture.md`, `security.md`, and `testing.md`: their frontmatter used a `globs:` key (a different tool's convention), which this repo's actual Claude Code loader does not read — confirmed by cross-checking against the 5 rule files that already worked correctly, all of which use `paths:`. Their existing glob lists were preserved, just renamed to `paths:`, so their intended scoping now actually takes effect instead of silently loading unconditionally.
- **Labeled** `constitutional-agent-engineering-rules.md`, `README.md`, and `surgical-density.md` as intentionally global via `paths: ["**/*"]`, matching the convention already established by `00-core-workflow.md`.
- **Rewrote** 14 lines of negative-phrasing ("do not", "never", "avoid", "cannot") across `CLAUDE.md` and 9 rule files into affirmative imperatives with measurable completion criteria, per the audit's linguistic-architecture axis.
- **Deferred** CCA-011 (MCP server governance — `allowedMcpServers`/`deniedMcpServers`): the repo still relies on a local `graphiti-memory` MCP server in `.mcp.json`, and any broader policy should be confirmed before moving the allow/deny list into managed settings.
- **Resolved** CCA-012 (`disableSkillShellExecution`): no in-repo skills or commands currently use inline `!` blocks, so project settings now disable inline shell execution for user/project/plugin/additional-directory skill sources.
- **Cleaned up** the remaining speculative `tools.*` settings keys and pinned `.claude/settings.json` to the real Claude Code settings schema URL. Branch-protection behavior is left to the repository's workflow rules and GitHub-side controls instead of pretending the project settings file can enforce it directly.
- **Confirmed not applicable**: CCA-025 (gitignore memory pattern) — this repo's local Claude auto-memory lives outside the repo at `~/.claude/projects/<hash>/memory/`; the only in-repo local artifact directory (`.claude/traces/`) is already gitignored.

### Residual risk

- `pre-tool-use-guard.js` is a heuristic path/command-pattern matcher, not an exhaustive parser — it can miss disguised paths (e.g. `>.env` with no space, or a path built via string concatenation in a script) and is defense-in-depth on top of `.claude/settings.json`'s native `permissions.deny` rules, not a substitute for reviewing diffs that touch Protected Areas.
- `stop.sh` verifies the deterministic control-plane checker; it still does not run any test, lint, or typecheck command because none are confirmed to exist in this repo.
- `constitutional-agent-engineering-rules.md` uses a `trigger: model_decision` frontmatter key not used anywhere else in this repo's rule files (this repo's real scoping mechanism is `paths:`, now added alongside it) and its content substantially overlaps with the user's global `operating-constitution.md`/`surgical-change-control.md` rules and this project's own `CLAUDE.md` constitution — worth a future consolidation pass, not done here since it was out of the approved scope for this batch.
