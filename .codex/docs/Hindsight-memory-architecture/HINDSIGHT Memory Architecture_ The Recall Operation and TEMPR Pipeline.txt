In the HINDSIGHT Memory Architecture, the Recall operation serves as the critical bridge between ingesting new information (Retain) and reasoning over it (Reflect) 1, 2. Managed by the TEMPR component, Recall is an agent-optimized retrieval mechanism designed to pull relevant context from the memory bank in response to a user's query 3, 4.
Unlike conventional search systems that return a fixed "top-$k$" number of results, HINDSIGHT's Recall operation is driven by a strict token budget ($k$) 5. This ensures the system retrieves exactly as much context as the downstream LLM's context window can handle, allowing the agent to request "just enough" memory for simple questions or expand its search budget for complex, multi-hop reasoning 6.
To populate this context budget, TEMPR executes a four-way parallel retrieval pipeline, fully leveraging the highly structured memory graph built during the Retain phase 7:
* Semantic Retrieval: Uses vector cosine similarity to capture conceptual matches and paraphrases 7, 8.
* Keyword Retrieval (BM25): Employs full-text lexical search for precise matching of specific terms or proper nouns, recovering information that might be ambiguous in the embedding space 8, 9.
* Graph Retrieval (Spreading Activation): Building on the entity-aware reasoning principle discussed earlier, this channel uses top semantic hits as starting points to traverse the memory graph. By propagating activation along entity, temporal, and causal links, it surfaces indirectly related facts that keyword or vector search alone would miss 9, 10.
* Temporal Graph Retrieval: Building on the temporal awareness principle, this channel uses a hybrid parser to convert temporal references in a query (e.g., "last weekend") into concrete date ranges. It then retrieves and prioritizes events whose actual occurrence intervals overlap with that timeframe 10, 11.
Once these four channels produce their candidate lists, they are merged using Reciprocal Rank Fusion (RRF) 12. Because RRF is rank-based rather than score-based, it does not rely on calibrated scores across the different systems; instead, it robustly elevates memories that receive multi-evidence support across the different search strategies 13. After fusion, a neural cross-encoder reranker refines the top candidates for maximum precision, incorporating temporal cues into its relevance scoring 13, 14.
As a final step, a Token Budget Filter greedily packs the text of these reranked memories until the specified token limit ($k$) is reached 14, 15. This perfectly sized, highly curated set of context is then passed to the CARA component for the final Core Operation—Reflect—where the agent uses this grounded evidence to generate a preference-conditioned response 16, 17.
