# Pending Self-Improvement Registry

This directory is reserved for machine-readable self-improvement lifecycle records.

- `registry.jsonl` records proposal states such as `proposed`, `applied`, `verified-kept`, `reverted`, `human-rejected`, and `deferred`.
- Runtime registry files may contain trace-derived task summaries and should be reviewed for sensitive content before committing.
- Proposals that mutate Tier 0 or Tier 2 surfaces should include an ID that can be linked to commits and rollback records.

Keep this README tracked so the registry location is discoverable even when no pending proposals exist.
